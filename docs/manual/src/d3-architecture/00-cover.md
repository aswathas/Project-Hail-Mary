# About this document

This is the architectural reference for ChainSentinel. It explains **what**
the system is built from, **how** data flows through it, and **why** the
key design choices were made. It assumes the reader is technical (engineer,
senior operator, or security architect) but does not require prior
familiarity with the code.

Companion documents:

- **D4 Detection Reference** — every signal, pattern, derived event, ABI,
  and ES field, in catalogue form.
- **D5 Developer Guide** — per-module API contracts and extension points.
- **D6 Operations Guide** — install, configure, deploy, troubleshoot.
- **D1 Executive Overview** — non-technical summary for stakeholders.
- **D2 Analyst Handbook** — workflow guide for SISA analysts.
- **D7 Simulation Guide** — the five Foundry attack scenarios.
