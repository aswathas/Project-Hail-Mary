# 6. Scenario: `demo-500`

## 6.1 Purpose

`demo-500` is a **narrative scenario** designed for live presentations.
It compresses 500 transactions into a single sequence that includes:

- ~300 legitimate user transactions (deposits, swaps, transfers).
- A reentrancy exploit against `VulnerableVault`.
- A sandwich attack against `SimpleDEX`.

A single `forge` invocation produces the full chain.

## 6.2 Contracts

| File | Role |
|------|------|
| `MockERC20.sol` | Base asset, shared with `simulations/shared/contracts/` |
| `VulnerableVault.sol` | Reentrancy target |
| `SimpleDEX.sol` | Sandwich target |
| `ReentrancyAttacker.sol` | Performs the recursive `withdraw` |
| `SandwichAttacker.sol` | Performs the bracket swap |

## 6.3 Script

`RunAll.s.sol` orchestrates all 500 transactions in one go. The
`docker-compose.yml`'s `simulator` service runs exactly this script
against the `anvil` service:

```yaml
simulator:
  ...
  command:
    - cd /sim && forge script scenarios/demo-500/script/RunAll.s.sol
      --rpc-url http://anvil:8545
      --broadcast
      --private-key 0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
      -vv
```

## 6.4 Expected output

The pipeline produces both reentrancy and sandwich alerts:

| Layer | Document |
|-------|----------|
| Alert | `AP-001 classic_reentrancy` |
| Alert | `AP-016 sandwich_attack` |
| Alert | `AP-030 attacker_deployment` |
| Alert | `AP-031 fund_dispersion` (if the attacker dispersed funds) |
| Case verdict | Multi-incident verdict; the copilot's Executive Summary breaks them out separately. |
